# نظام نقاط الطلاب والمجموعات

نظام بسيط يعمل في المتصفح مباشرة (بدون أي تثبيت) لإدارة النقاط الفردية للطلاب،
ويحسب مجموع نقاط كل مجموعة تلقائيًا، مع شاشة عرض كبيرة تتحدّث لحظيًا.

## التشغيل

1. افتح ملف `index.html` (بوابة الدخول الموحدة)، اختر اسمك وأدخل كلمة المرور.
2. بعد الدخول ستنتقل إلى `dashboard.html` التي تعرض الأقسام المناسبة لصلاحية الحساب.
3. لشاشة العرض الكبيرة، افتح `display.html` (أو من بطاقة شاشة المجموعات)
   واجعلها بملء الشاشة عبر زر ملء الشاشة ⛶ أو F11 على البروجكتر.

> البيانات تتزامن لحظيًا عبر كل الأجهزة (Firebase) — راجع قسم المزامنة بالأسفل.

### داخل لوحة البرنامج (`admin.html`)

⚡ إضافة نقاط (مع أزرار سريعة وأسباب جاهزة وتراجع) • 👥 إدارة الطلاب (فردي + دفعة) •
📋 النقاط الفردية (فلترة وترتيب) • 🏆 المجموعات • 📊 التقارير والإحصائيات •
🧾 سجل العمليات (مع تراجع) • ⚙️ الإعدادات.

### إدارة الحسابات (`owner.html`)

- تظهر لمالك المنصة فقط.
- إضافة حساب معلم أو إدارة وتحديد مرحلته.
- تفعيل الحساب أو إيقافه وتعديل كلمة مروره.
- منح صلاحيات التحضير والإغلاق وإدارة الطلاب والتقارير وشاشات العرض.
- حساب مالك المنصة محمي من الحذف أو الإيقاف أو تغيير الرتبة.
- لكل حساب معرّف ثابت داخلي؛ لذا تعديل اسم معلم **لا يُخرج جلسته** على بقية الأجهزة،
  بل يُحدَّث الاسم المعروض تلقائيًا مع بقاء صلاحياته.

### تحضير الثانوية (`attendance-high.html`)

- يظهر لمعلمي الثانوية والإدارة والمالك بحسب المرحلة والصلاحية.
- حالات التحضير: مبكر، حاضر، غائب، وغير محدد.
- بحث وتصفية وتحديد جماعي للطلاب.
- ملخص لحظي ونسبة اكتمال للتحضير.
- إغلاق واعتماد اليوم مع صلاحية خاصة لإعادة فتحه.
- إدارة مستقلة لطلاب الثانوية دون التأثير في نقاط أو مجموعات المتوسطة.

### بوابة الإدارة العامة (`admin-portal.html`)

- متابعة أعداد ونسب حضور المرحلتين لحظيًا.
- تقرير موحد حسب التاريخ والمرحلة والحالة والمعلم.
- طباعة التقرير وتصديره بصيغة CSV.
- إرسال توجيه عادي أو عاجل لجميع المعلمين أو لمرحلة محددة.
- يظهر أحدث توجيه نشط تلقائيًا أعلى صفحة التحضير المستهدفة.

## التحضير

- لكل **تاريخ** تحضير مستقل؛ تحدّد حالة الطالب فتُمنح نقاط الحالة تلقائيًا.
- تغيير حالة طالب في نفس اليوم **يعدّل النقاط دون تكرار** (يسحب القديمة ويطبّق الجديدة).
- قيم النقاط لكل حالة تُضبط من: الإعدادات ← **نقاط التحضير**.
- عمليات التحضير تظهر في السجل (تُصحَّح من صفحة التحضير لا من السجل).
- «بدء جولة جديدة» (تصفير النقاط) يصفّر التحضير أيضًا.

## القواعد المطبّقة

- نقاط المجموعة = مجموع نقاط طلابها (لا تُخزّن يدويًا).
- لا يمكن أن تقل نقاط الطالب عن صفر (الخصم يتوقف عند الصفر).
- نسبة الشريط = النقاط ÷ الهدف × 100، ويبقى ممتلئًا عند تجاوز الهدف مع عرض العدد الحقيقي.
- كل عملية تُسجَّل: الطالب، المجموعة، العدد، النوع، السبب، الوقت، والمشرف.

## البيانات والمزامنة (Firebase)

النظام يحفظ نسخة محلية (`localStorage`) للعمل السريع/دون اتصال، **ويتزامن لحظيًا
عبر كل الأجهزة** من خلال Firebase Firestore. بيانات المتوسطة تستخدم الكولكشنات
الحالية (`students` و`attendance` و`logs`)، وبيانات الثانوية معزولة تحت
`stages/high/students` و`stages/high/attendance`، بينما تحفظ التوجيهات في `memos`.

### خطوات تفعيل Firebase (تُعمل مرة واحدة من لوحة Firebase)

1. ادخل [Firebase Console](https://console.firebase.google.com/) → مشروع `rhhal-6de3d`.
2. **Build → Firestore Database → Create database** → اختر **Start in test mode**
   (أو production ثم اضبط القواعد بالأسفل) → اختر المنطقة → Enable.
3. **قواعد الأمان (Rules)** — للبدء السريع (غير آمنة للعموم، تكفي للتجربة):

   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /{document=**} {
         allow read, write: if true;
       }
     }
   }
   ```

   > ⚠️ هذه القواعد تسمح للجميع بالقراءة والكتابة. لاحقًا يُنصح بتقييدها
   > (مثلاً عبر تسجيل دخول للمشرفين، أو جعل شاشات العرض للقراءة فقط).

### تشغيله على عدة أجهزة

لأن المشرف وشاشة العرض على أجهزة مختلفة، كل جهاز يحتاج الوصول لملفات الموقع:

- **الأسهل (استضافة):** ارفع الملفات على استضافة ثابتة مجانية تعطيك رابطًا واحدًا
  يفتحه الجميع — مثل **Netlify Drop** (اسحب المجلد فقط)، أو **GitHub Pages**،
  أو **Firebase Hosting** (يحتاج Node CLI).
- **بدون استضافة:** انسخ مجلد المشروع إلى كل جهاز وافتح الملف محليًا — ستظل كل
  الأجهزة تتزامن عبر Firestore طالما لديها إنترنت.

> ملاحظة: النقاط تُكتب كتغيّر ذرّي (`FieldValue.increment`) لا كقيمة مطلقة، وسجل
> تحضير كل طالب يُكتب عبر مسار حقله وحده. لذا يمكن لعدة مشرفين رصد النقاط والتحضير
> في نفس اللحظة دون أن تدهس عملية أخرى أو تُفقد. (تحضير الثانوية يستخدم معاملة
> `runTransaction` كذلك.)
>
> استثناء بسيط: قاعدة «لا تقل النقاط عن صفر» تُطبَّق محليًا؛ في حال خصمين متزامنين
> يهبطان بالطالب تحت الصفر قد تظهر قيمة سالبة مؤقتة تُصحَّح بأي عملية إضافة لاحقة.

## النسخ الاحتياطي والاستعادة

- من لوحة المالك يمكن تنزيل ملف JSON موحد بإصدار بنية `2.0`.
- تشمل النسخة الحسابات وطلاب المرحلتين والتحضير والنقاط والسجل والتوجيهات.
- يفحص النظام بنية الملف ويعرض ملخص محتواه قبل السماح بالاستعادة.
- ينزّل النظام نسخة وقائية من البيانات الحالية قبل بدء الاستعادة.
- الاستعادة الكاملة متاحة لمالك المنصة فقط، وتستبدل الكولكشنات على دفعات لتجنب
  الحدود الحجمية ولإزالة البيانات القديمة غير الموجودة في النسخة.
- ما زالت النسخ القديمة من النظام مدعومة عند الاستيراد.

## سجل العمليات وترحيل بيانات المتوسطة

- تسجل لوحة المالك العمليات الحساسة مثل إدارة الحسابات، إغلاق التحضير، التوجيهات، الاستعادة، والترحيل.
- أداة ترحيل المتوسطة تنسخ `students` و`attendance` و`logs` إلى الهيكل المنظم تحت `stages/middle`.
- لا تحذف الأداة الجداول القديمة ولا تغيّرها، ويمكن تشغيل التحقق بصورة مستقلة.
- يقارن التحقق أعداد السجلات ومجموع النقاط ومعرّفات ومحتوى الوثائق، وليس الأعداد وحدها.
- قبل بدء الترحيل تنزّل لوحة المالك نسخة احتياطية تلقائياً.
- راجع `قائمة-الإطلاق.md` لتسلسل التشغيل والتراجع عند وجود اختلاف.
